\section{Introduction}
The management of technical events has become increasingly complex with growing attendance and diverse event types. Traditional methods of event booking through manual registration or email often lead to inefficiencies, double bookings, and poor user experience. The Tech Event Ticket Booking system addresses these challenges by providing a digital platform that simplifies event discovery, ticket booking, and administrative management.

This system serves as a centralized hub where employees and students can view upcoming events, filter by categories or tags, book tickets, and receive real-time notifications about their bookings and new events. Administrators gain the ability to create events, manage attendee lists, track ticket availability, and communicate with users through the notification system.

The application leverages modern web technologies to ensure scalability, real-time communication, and an intuitive user interface. The integration of WebSocket technology enables instant notifications, while the tag-based categorization system improves event discoverability.

Eventbrite is a leading global platform for event management and ticketing. It offers comprehensive event discovery, ticket sales, and attendee management features. However, it is designed for public events and lacks customization for institution-specific technical event requirements.

\textbf{2. Meetup (https://www.meetup.com/)}
Meetup focuses on community event organization and social networking around events. While it provides good event discovery features, it is not designed for academic or technical event workflows with administrative controls.

\textbf{3. Ticketmaster (https://www.ticketmaster.com/)}
Ticketmaster specializes in large-scale entertainment events and ticketing. It is enterprise-level but not suitable for smaller technical events due to complexity and cost.

\textbf{4. Eventsquid (https://www.eventsquid.com/)}
Eventsquid provides event management for conferences and meetings. It includes features like attendee management and reporting but lacks real-time notification capabilities.

// TagFilter.jsx - Event filtering by tags
import React, { useState, useEffect } from 'react';
import { tagService } from '../services/tagService';

const TagFilter = ({ onTagsChange, selectedTags = [] }) => {
  const [tags, setTags] = useState([]);

  useEffect(() => {
    const loadTags = async () => {
      const tagsData = await tagService.getAllTags();
      setTags(tagsData);
    };
    loadTags();
  }, []);

  const handleTagToggle = (tagId) => {
    const newSelectedTags = selectedTags.includes(tagId)
      ? selectedTags.filter((id) => id !== tagId)
      : [...selectedTags, tagId];
    onTagsChange(newSelectedTags);
  };

  return (
    <div className="tag-filter mb-3">
      <label className="form-label fw-bold">Filter by Tags:</label>
      <div className="d-flex flex-wrap gap-2">
        {tags.map((tag) => (
          <button
            key={tag.id}
            className="btn btn-sm"
            onClick={() => handleTagToggle(tag.id)}
            style={{ backgroundColor: tag.color }}
          >
            {tag.name}
          </button>
        ))}
      </div>
    </div>
  );
};

// server.js - WebSocket initialization
const http = require("http");
const app = require("./app");
const { Server } = require("socket.io");

const server = http.createServer(app);

const io = new Server(server, {
  cors: {
    origin: "http://localhost:5173",
    credentials: true
  }
});

// Store connected users
const connectedUsers = new Map();

io.on("connection", (socket) => {
  console.log(`User connected: ${socket.id}`);

  // User joins their notification room
  socket.on("join_notifications", (userId) => {
    socket.join(`user_${userId}`);
    connectedUsers.set(userId, socket.id);
  });

  socket.on("disconnect", () => {
    for (let [userId, socketId] of connectedUsers.entries()) {
      if (socketId === socket.id) {
        connectedUsers.delete(userId);
        break;
      }
    }
  });
});

app.locals.io = io;
server.listen(5000, () => {
  console.log("Server running on http://localhost:5000");
});
\end{lstlisting}

\textbf{3.3.3 Tag Model Implementation}

Database operations for tags:

\begin{lstlisting}
// tagModel.js - Tag database operations
async function getAllTags() {
  const query = `
    SELECT id, name, color, created_at
    FROM tags
    ORDER BY name
  `;
  const [tags] = await pool.query(query);
  return tags;
}

async function addTagToEvent(eventId, tagId) {
  const query = `
    INSERT IGNORE INTO event_tags (event_id, tag_id)
    VALUES (?, ?)
  `;
  const [result] = await pool.query(query, [eventId, tagId]);
  return result.affectedRows > 0;
}

async function getEventTags(eventId) {
  const query = `
    SELECT t.id, t.name, t.color
    FROM tags t
    INNER JOIN event_tags et ON t.id = et.tag_id
    WHERE et.event_id = ?
  `;
  const [tags] = await pool.query(query, [eventId]);
  return tags;
}

async function updateEventTags(eventId, tagIds) {
  await pool.query(
    "DELETE FROM event_tags WHERE event_id = ?", 
    [eventId]
  );
  
  for (const tagId of tagIds) {
    await addTagToEvent(eventId, tagId);
  }
  return true;
}

\section{Project Architecture} 
  
The application follows a three-tier architecture pattern:

\begin{itemize}
\item \textbf{Presentation Tier:} React.js frontend with responsive UI components
\item \textbf{Business Logic Tier:} Node.js/Express.js REST API with authentication
\item \textbf{Data Tier:} MySQL database with normalized schema
\item \textbf{Real-time Communication:} Socket.io WebSocket layer for notifications
\end{itemize}

Authentication flow uses JWT tokens issued on login and verified on protected routes. The WebSocket connection establishes user-specific notification rooms for targeted message delivery.

\section{Data Flow Diagram} 

The application flow follows this pattern:

\begin{enumerate}
\item User registers/logs in via authentication API
\item JWT token is issued and stored in browser
\item WebSocket connection established with user ID
\item User browses events filtered by category/tags
\item User books tickets through booking API
\item Backend creates notification record and emits via WebSocket
\item Notification appears in real-time in notification center
\item User can mark notifications as read or delete
\end{enumerate}

\section{Module 1: Authentication Module} 
Handles user registration, login, and JWT token management:
\begin{itemize}
\item User registration with email validation
\item Password hashing with bcryptjs
\item JWT token generation on login
\item Role-based access control (User/Admin)
\item Protected route middleware for API endpoints
\end{itemize}

\section{Module 2: Event Management Module} 
Core event operations including creation, listing, and filtering:
\begin{itemize}
\item Create events (admin only)
\item List events with pagination
\item Filter by category, date, venue, and tags
\item Update event details
\item Delete events
\item Track available seats
\item Event details with associated tags
\end{itemize}

\section{Module 3: Notification and Real-time Communication Module} 
WebSocket-based real-time notifications:
\begin{itemize}
\item WebSocket connection management with Socket.io
\item User-specific notification rooms
\item Booking confirmation notifications
\item Event creation announcements
\item Real-time notification delivery
\item Notification history storage
\item Mark as read/unread functionality
\item Notification deletion
\end{itemize}

\section{Module 4: Tag-based Categorization Module}
Event categorization using flexible tag system:
\begin{itemize}
\item Create and manage tags with color coding
\item Assign multiple tags to events
\item Filter events by selected tags
\item Tag-based event discovery
\item Admin panel for tag management
\item Tag suggestions based on event type
\end{itemize}

\textbf{Advantages of this Project} 

\textbf{a) Real-time Communication:} 
WebSocket integration enables instant notifications for users about booking confirmations and new events, improving user engagement and reducing response time.

\textbf{b) Flexible Tag System:} 
The tag-based categorization allows users to discover events based on their interests rather than rigid categories, enabling better event discoverability.

\textbf{c) Scalable Architecture:} 
Modular design and separation of concerns allow easy addition of new features and scaling of components independently.

\textbf{d) Security:} 
JWT-based authentication with role-based access control ensures secure user data and admin-only operations.

\textbf{e) User Experience:} 
Responsive design with real-time feedback provides an intuitive and engaging user interface.

\textbf{Disadvantages} 

\begin{itemize}
\item WebSocket connections require persistent server resources
\item Initial setup and deployment complexity higher than simple CRUD applications
\item Database optimization required for large-scale event and user data
\item Real-time features may be overkill for small organizations
\end{itemize}
