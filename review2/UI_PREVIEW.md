# UI Preview Description

## Home Page
- A gradient hero section introduces the platform.
- Responsive event cards show image, title, short description, date, venue, and available seats.
- Filter panel includes search, venue filter, date range, and sorting.
- Pagination controls at the bottom (`Previous`/`Next`).

## Event Details Page
- Large event image on the left, event information card on the right.
- Includes date, venue, seat availability, and direct booking button.

## Ticket Booking Page
- Booking card with event summary and ticket count input.
- Client-side validation for min/max ticket count and available seats.
- Booking confirmation alert displays booking ID and remaining seats.

## Login/Signup Pages
- Card-based forms with clean spacing and validation feedback.
- Login redirects to requested protected route.

## Dashboard
- User-specific booking cards with event image and booking details.
- Handles empty state when no bookings exist.

## Admin Panel
- Left side: create/update event form.
- Right side: event table with Edit/Delete actions.
- Built for quick internal department management.

## Responsive Behavior
- Navbar collapses on mobile.
- Cards stack in one column on small screens and grid on larger screens.
- Forms and tables remain usable on narrow viewports.
